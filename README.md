Food Order Management System
Overview
The Food Order Management System is a web-based application built using ASP.NET Web Forms and Oracle Database. It was designed to digitize and streamline internal food ordering operations within an organization. The system replaces manual processes with a centralized platform for order placement, tracking, and administrative management.

Features
1.Role-based Login Authentication – Secure access for employees and administrators.
2.Employee Dashboard – view, and track/approve food orders.
3.Admin Dashboard – Monitor all orders and manage system operations.
4.Food Item Management – Add, update, or remove menu items efficiently.
5.Centralized Order Records – Maintain a reliable database of all orders.
6.Session Handling and Access Control – Ensures secure and authorized use of the system.

System Architecture
The application follows a three-tier architecture for maintainability and separation of concerns:
1.Presentation Layer – Built with ASP.NET Web Forms to provide an interactive user interface.
2.Business Logic Layer – Implemented in C# to handle backend processes and application logic.
3.Data Layer – Oracle Database manages all data storage, retrieval, and integrity.

Technologies Used

1.ASP.NET Web Forms – Frontend interface development

2.C# – Backend logic and business layer
3.Oracle Database – Data storage and management
4.HTML & CSS – Web interface design
5.Visual Studio – Development environment
6.Git & GitHub – Version control and code management

Project Objective
The main goal of this project was to develop a structured and secure web application that improves operational efficiency, reduces manual errors, and provides reliable record-keeping for internal food ordering. By automating the order process, the system helps organizations save time and ensures smooth operations.
